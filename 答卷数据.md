## 数据结构设计

回顾：问卷VS答卷

一个问卷发布了，用户提交的一份**答卷**，数据结构如下：

```js
{
    questionId:'500000201704276152',
    answerList:[
        {componentId:'c1',value:undefined},   //标题info
        {componentId:'c2',value:'张三'},      //input
        {componentId:'c3',value:'13235613935'},    //input
        {componentId:'c4',value:'1-3年'},       //单选
        {componentId:'c5',value:'Vue2,Vue3'},   //多选
        {componentId:'c6',value:'大于20k'},     //单选
        {componentId:'c7',value:'备注xx'}       //textarea
    ]
}

```

**一份问卷对应多份答卷**，所以一份问卷的所有答卷列表如下：
```js
[
    //用户A
    {
        _id:'xx',
        c2:'张三',
        c3:'13235613935',
        c4:'1-3年',
        c5:'Vue2,Vue3',
        c6:'大于20k',
        c7:'备注xx'
    },
    //用户B
    {
        _id:'yy',
        c2:'李四',
        c3:'13235613935',
        c4:'3-5年',
        c5:'Vue2,React',
        c6:'10-20k',
        c7:'备注yy'
    }
    //更多用户
]

````
